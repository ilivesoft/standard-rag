# Product: Local RAG Pipeline

## 프로젝트 개요

**Local RAG Pipeline**은 Windows 환경에서 로컬로 실행 가능한 데모용 End-to-End RAG(Retrieval-Augmented Generation) 파이프라인 시스템입니다.

사용자가 문서(파일 또는 폴더)를 업로드하면 자동으로 파싱, 정제, 청킹, 임베딩 생성부터 하이브리드 검색, 리랭킹, LLM 응답 생성까지 전체 파이프라인을 처리합니다.

---

## 핵심 기능

### 문서 처리 파이프라인

| 단계 | 기능 | 설명 |
|------|------|------|
| 1 | 문서 파싱 | PDF, DOCX, TXT, Markdown, HTML 등 다양한 형식 자동 파싱 |
| 2 | 텍스트 정제 | 특수문자 제거, 중복 공백 처리, 노이즈 텍스트 정규화 |
| 3 | Chunk 분할 | 의미 단위 청킹 (RecursiveCharacterTextSplitter, 오버랩 지원) |
| 4 | Embedding 생성 | 로컬 sentence-transformers 또는 OpenAI Embeddings |
| 5 | Vector DB 저장 | ChromaDB에 벡터 및 메타데이터 영속 저장 |
| 6 | 하이브리드 검색 | Vector 유사도 + BM25 키워드 검색 병합 |
| 7 | Reranker 정렬 | Cross-encoder 기반 재순위 정렬로 검색 품질 향상 |
| 8 | LLM 응답 생성 | 검색된 컨텍스트 기반 자연어 응답 생성 |

### 인터페이스

- **Web UI**: Gradio 기반 데모 인터페이스 (다중 탭 지원)
  - 문서 업로드 탭
  - 질의응답 탭 (채팅 형식)
  - 대화 이력 탭
  - 설정 관리 탭
- **REST API**: FastAPI 기반 프로그래매틱 접근
  - 문서 인덱싱 엔드포인트
  - 질의응답 엔드포인트
  - 대화 관리 엔드포인트 (NEW)
  - 컬렉션 관리 엔드포인트
- **폴더/파일 업로드**: 단일 파일 및 폴더 일괄 처리 지원
- **OCR 지원**: 스캔 이미지 PDF 자동 처리 (NEW)

## 대상 사용자

- RAG 파이프라인을 이해하고 싶은 개발자 및 AI 엔지니어
- 로컬에서 문서 기반 QA 시스템을 빠르게 프로토타이핑하고 싶은 사용자
- LLM 애플리케이션 데모가 필요한 기술 세일즈 및 교육 담당자

---

## 사용 시나리오

### 시나리오 1: 기술 문서 QA

1. 제품 매뉴얼 PDF 폴더를 업로드
2. 시스템이 자동으로 파싱 → 청킹 → 임베딩 저장
3. "이 제품의 전원을 끄는 방법은?" 질문
4. 하이브리드 검색 → Reranker → LLM이 정확한 섹션 기반 답변 생성

### 시나리오 2: 사내 지식베이스 검색

1. 사내 보고서, 정책 문서 업로드
2. 자연어로 정책 질의
3. 관련 문서 출처와 함께 답변 제공

## 제약 사항

- Windows 로컬 실행 환경 (CPU 기반도 동작, GPU 권장)
- 데모용 (프로덕션 보안/스케일링 미포함)
- 최대 파일 크기: 100MB (데모 기준)
- 지원 언어: 한국어, 영어 (다국어 임베딩 모델 사용 시 확장 가능)
